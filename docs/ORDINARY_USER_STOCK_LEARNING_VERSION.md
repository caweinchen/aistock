# 普通用户股票学习与风险识别版本规划

## 1. 背景判断

当前 App 更接近一个股票数据与策略分析工具，包含行情、因子评分、策略回测、风险提示、分红、新闻、复权因子、重要持有人数据等模块。它对有一定投资基础的用户有帮助，但对普通用户并不一定友好。

普通用户的核心问题通常不是“缺少数据”，而是：

- 看不懂指标含义。
- 容易把评分或信号当成直接买卖建议。
- 不知道风险来自哪里。
- 不知道下一步应该观察什么。
- 容易被短期新闻、涨跌幅或单一指标影响判断。
- 不知道如何在多只自选股之间做取舍。

因此，普通用户版本不应定位为“荐股工具”，而应定位为：

> 面向普通投资者的股票学习、风险识别与自选股决策参考助手。

核心目标是帮助用户逐步学会股票分析，识别明显风险，并在自己的自选股票池中找到更值得关注的对象。

## 2. 新版本目标

普通用户版本的主要目标：

1. 帮助用户快速理解一只股票的基本情况。
2. 帮助用户学习买卖股票前应该看哪些因素。
3. 帮助用户识别估值、波动、业绩、资金、股东变动等风险。
4. 帮助用户在自选股池中做专业化筛选和排序。
5. 帮助用户形成检查清单式的投资决策流程。
6. 在风险可控的前提下，提高普通用户获得更好收益的概率。

需要避免的方向：

- 不直接给出强烈的“买入/卖出”指令。
- 不把历史回测包装成未来收益承诺。
- 不用单一分数决定股票好坏。
- 不把数据不足的情况伪装成确定结论。
- 不把自选股排序包装成确定性收益推荐。

## 3. 当前 App 对普通用户的主要不足

### 3.1 买卖信号过于直接

当前系统中存在 `buy`、`sell`、`neutral` 这类信号。普通用户可能会把它理解成明确荐股建议。

更适合普通用户的表达方式是：

- 偏积极
- 中性观察
- 偏谨慎
- 数据不足，暂不判断

这样能降低用户误用系统的风险。

### 3.2 评分缺少足够解释

普通用户看到 80 分、60 分、40 分，很难理解分数来自哪里。

建议将评分拆成：

- 估值水平
- 盈利能力
- 成长情况
- 价格趋势
- 资金关注度
- 股东变动
- 波动风险
- 数据完整度

每个评分都要有一句普通语言解释。例如：

> 当前估值高于历史常见区间，如果后续业绩不能继续增长，股价可能承压。

### 3.3 自选股只是列表，没有决策参考能力

普通用户通常会把感兴趣的股票加入自选，但真正需要的是：

- 哪些自选股值得重点关注？
- 哪些自选股风险更高？
- 哪些股票只是因为短期上涨吸引注意？
- 哪些股票数据不足，暂时不适合判断？
- 自选股之间如何比较？

当前 App 的自选股更像行情列表，还没有形成“自选股池分析”和“机会/风险排序”的能力。

### 3.4 “机构持仓”语义需要调整

TuShare `top10_holders` 更准确的含义是“前十大股东”或“重要持有人”，不等同于严格机构持仓。

对普通用户来说，更有价值的不是“是不是机构”，而是：

- 谁在持有？
- 持股比例是多少？
- 最近是增持还是减持？
- 变动是否明显？
- 这是否说明筹码稳定或存在减持压力？

建议模块命名改为：

- 重要股东变动
- 前十大股东持仓
- 重要持有人趋势

展示重点应从“机构类型”转向“持仓变化”。

### 3.5 新闻模块容易误导

普通用户容易被新闻标题影响，尤其是市场快讯类新闻。

如果没有可靠的个股重大事项接口，不建议强行展示“个股新闻”。更好的方向是展示“重大事件”：

- 业绩预告
- 分红
- 股东减持
- 限售解禁
- 监管处罚
- 并购重组
- 高管变动

如果只能获取市场新闻，应明确标注为“市场新闻”，不要暗示它直接关联该股票。

### 3.6 回测结果需要解释适用场景

普通用户看到策略收益率、胜率、最大回撤，容易误以为未来也会这样。

建议增加解释：

- 这个策略适合什么市场环境。
- 什么时候容易失效。
- 最大回撤代表用户可能亏多少。
- 胜率和收益率如何一起理解。
- 当前是否真的满足策略条件。

回测模块应强调：历史表现不是未来收益保证。

## 4. 新版本核心能力：自选股池智能参考

普通用户版本最值得优先建设的核心功能，是基于用户自选股票池，利用本地已有分析结果，输出更专业但可解释的参考排序。

这比“全市场荐股”更适合当前项目：

- 用户已经对这些股票有兴趣，参考结果更贴近真实决策。
- 自选池范围较小，更适合本地缓存和本地计算。
- 可以减少外部接口访问，主要复用本地行情、历史、因子、风险、策略、分红、重要股东数据。
- 输出可以做成“关注建议”和“风险提醒”，避免直接荐股。

建议功能名称：

- 自选股机会雷达
- 自选股健康度
- 今日观察建议
- 重点关注清单
- 自选股风险排序

不建议命名为：

- 推荐买入
- 必买股票
- 最佳买点
- 今日荐股

## 5. 自选股池推荐参考框架

### 5.1 输出分组

自选股池不要只给一个总排名，而应分组输出。

#### 重点关注

适合放入重点关注的股票通常具备：

- 综合表现较好。
- 数据完整度较高。
- 估值不过高。
- 价格趋势改善。
- 风险提示较少。
- 重要股东持仓稳定或增持。
- 回测或策略信号没有明显恶化。

展示语气示例：

> 当前在你的自选股中综合表现较好，适合加入重点观察列表。但仍需结合仓位和估值，不建议仅凭排名买入。

#### 继续观察

适合继续观察的股票通常具备：

- 基本面尚可。
- 趋势不够明确。
- 估值或资金信号一般。
- 暂无明显高风险，但优势也不突出。

展示语气示例：

> 当前缺少明确优势，可继续观察后续业绩、资金和价格趋势变化。

#### 谨慎关注

需要谨慎的股票通常存在：

- 估值风险高。
- 波动过大。
- 资金流出。
- 重要股东减持。
- 回测表现较弱。
- 风险提示较多。

展示语气示例：

> 当前风险项较多，不适合作为普通用户的优先选择。建议先确认风险是否可接受。

#### 数据不足

数据不足的股票不应强行给结论。

常见原因：

- 历史行情不足。
- 财务因子缺失。
- 重要股东数据缺失。
- 回测无法形成有效结果。
- 缓存数据过旧。

展示语气示例：

> 当前数据不足，暂不适合形成明确判断。建议补充数据后再分析。

### 5.2 推荐参考评分维度

建议评分不要只保留一个总分，而是拆成多个可解释维度：

| 维度 | 含义 | 对普通用户的解释 |
|---|---|---|
| 估值合理性 | 当前价格是否偏贵 | 避免在明显高估时追高 |
| 盈利质量 | 公司赚钱能力是否稳定 | 判断公司基本面是否可靠 |
| 成长变化 | 收入和利润是否改善 | 判断未来预期是否变好 |
| 趋势状态 | 价格走势是否改善 | 判断市场是否开始认可 |
| 波动风险 | 股价波动是否过大 | 判断用户能否承受回撤 |
| 资金关注 | 资金流向是否积极 | 判断短期资金态度 |
| 重要股东变动 | 重要持有人是否增减持 | 判断筹码稳定性 |
| 分红稳定性 | 是否有稳定回报 | 辅助判断长期持有价值 |
| 策略适配度 | 回测策略是否适用 | 判断当前是否满足策略条件 |
| 数据完整度 | 本地数据是否足够 | 防止用不完整数据下结论 |

### 5.3 参考排序逻辑

推荐采用“规则评分 + 解释生成”的方式。

规则评分用于稳定排序：

- 估值合理：加分。
- 盈利能力较好：加分。
- 趋势改善：加分。
- 分红稳定：加分。
- 重要股东增持或稳定：加分。
- 风险提示较多：扣分。
- 波动过高：扣分。
- 重要股东减持：扣分。
- 数据缺失：扣分。
- 回测最大回撤过大：扣分。

解释生成用于让普通用户理解结果：

> 在你的自选股中，A 股票当前综合表现靠前，主要原因是盈利能力稳定、价格趋势较强、风险提示较少。但估值不算便宜，适合继续观察，不适合盲目追高。

### 5.4 不应输出的内容

自选股智能参考不应输出：

- “立即买入”
- “一定上涨”
- “稳赚”
- “满仓”
- “无风险”
- “最佳买点已出现”

更合适的表达：

- “适合重点观察”
- “当前偏积极”
- “风险较多，建议谨慎”
- “数据不足，暂不判断”
- “适合进入操作前检查”

## 6. 推荐页面规划

### 6.1 首页：今日自选股参考

首页可以直接展示用户最需要的信息：

- 今日重点关注
- 今日谨慎关注
- 数据不足股票
- 自选股整体风险
- 最近数据更新时间

示例结构：

```text
今日自选股参考

重点关注：
1. A 股票：趋势改善，风险较少，但估值略高。
2. B 股票：盈利稳定，波动较低，适合中长期观察。

谨慎关注：
1. C 股票：近期波动偏高，重要股东减持。
2. D 股票：数据不足，暂不形成判断。

操作前提醒：
- 不建议只因为排名靠前就买入。
- 先确认仓位、估值和可承受回撤。
- 单只股票不建议占比过高。
```

### 6.2 自选股健康度列表

每只自选股展示：

- 当前参考状态：偏积极 / 观察 / 谨慎 / 数据不足
- 总体健康度
- 最大风险标签
- 最大支撑因素
- 数据更新时间

排序方式：

- 按综合参考分排序。
- 按风险优先排序。
- 按数据完整度排序。
- 按最近变化排序。

### 6.3 个股详情页：普通用户摘要

每只股票详情页顶部展示：

- 一句话结论。
- 三个主要支撑点。
- 三个主要风险点。
- 数据更新时间。
- 数据完整度。

示例：

> 当前整体偏中性，价格趋势较强，但估值不低，且需要关注重要股东持仓变化。适合继续观察，不宜只根据短期上涨追高。

### 6.4 操作前检查页

当用户点击“准备买入参考”或“准备卖出参考”时，不直接给买卖指令，而是进入检查清单。

买入前检查：

- 我是否了解公司主营业务？
- 当前估值是否偏高？
- 最近业绩是否稳定？
- 是否存在重要股东减持？
- 股价是否处于大幅上涨之后？
- 如果下跌 10%-20%，我是否能接受？
- 我的持仓是否已经过度集中在同一行业？

卖出/减仓检查：

- 买入逻辑是否已经失效？
- 业绩是否明显变差？
- 估值是否过高？
- 重要股东是否连续减持？
- 是否只是因为短期波动而恐慌？

系统可以输出“通过项”和“未通过项”，但不直接替用户下买卖命令。

## 7. 信息架构规划

### 7.1 学习式评分卡

将专业指标转成普通语言：

- 估值：便宜 / 合理 / 偏贵 / 明显偏贵
- 业绩：稳定 / 改善 / 下滑 / 波动较大
- 趋势：走强 / 横盘 / 走弱
- 波动：较低 / 正常 / 偏高
- 股东变动：稳定 / 增持 / 减持 / 数据不足
- 资金关注：较强 / 一般 / 较弱

每项都附带“为什么”。

### 7.2 风险雷达

风险提示应分层：

- 高风险：需要重点注意，可能影响投资决策。
- 中风险：需要跟踪，但不一定立即否定。
- 低风险：当前影响较小。

风险类型包括：

- 估值风险
- 业绩风险
- 波动风险
- 资金流出风险
- 股东减持风险
- 数据不足风险
- 行业集中风险

### 7.3 重要股东变动

基于 `top10_holders` 展示：

- 股东名称
- 股东类型
- 报告期
- 持股比例
- 持股变动
- 变化方向

普通解释示例：

> 前十大股东中出现明显减持，需要关注后续公告和股价压力。

或：

> 重要持有人持仓整体稳定，筹码结构暂未出现明显异常。

### 7.4 组合风险视角

普通用户往往更需要组合层面的帮助。

建议增加：

- 行业集中度。
- 单只股票占比。
- 高风险股票数量。
- 同类股票重复持仓。
- 整体波动风险。

例如：

> 当前自选股中银行类股票占比较高，如果银行板块整体下跌，组合可能同步承压。

## 8. 数据与接口建议

### 8.1 EastMoney

EastMoney 实时行情不受当前 TuShare Pro 访问限制影响，可以继续用于：

- 实时价格
- 涨跌幅
- 股票搜索
- 基础行情展示

### 8.2 TuShare Pro

TuShare Pro 数据应继续本地缓存优先，减少访问频率。

适合普通用户版本的数据：

- 历史行情
- 财务指标
- 分红
- 前十大股东
- 复权因子
- 重大事项类数据

不建议将不支持个股语义的新闻接口包装成个股新闻。

### 8.3 本地分析结果

普通用户版本应优先复用本地已有结果：

- `Stock`：价格、涨跌幅、基础评分、更新时间。
- `PricePointDB`：历史行情、趋势、波动、回测基础。
- `FactorScoreDB`：估值、动量、波动、盈利等因子。
- `StrategyResultDB`：策略回测摘要。
- `AlertItemDB`：风险提示。
- `DividendDB`：分红记录。
- `InstHoldDB`：重要持有人变动。
- `AdjFactorDB`：复权因子。

自选股池推荐参考应尽量使用本地数据计算，只在数据缺失或过期时触发外部数据刷新。

## 9. 推荐参考的安全边界

普通用户版本必须明确边界：

- 系统提供的是分析参考，不是投资建议。
- 排名靠前不代表一定上涨。
- 历史回测不代表未来收益。
- 数据不完整时必须降低结论强度。
- 所有操作前都应检查风险和仓位。

推荐文案：

- “当前偏积极，但仍需关注估值风险。”
- “数据不足，暂不适合形成明确判断。”
- “该信号只能作为辅助参考，不能单独作为买入依据。”
- “建议先观察业绩和股东变动是否继续改善。”
- “适合加入重点观察，不代表建议立即买入。”

避免文案：

- “强烈买入”
- “稳赚”
- “必涨”
- “最佳买点”
- “无风险”

## 10. 分阶段规划

### 第一阶段：降低误导风险

目标：让当前 App 更适合普通用户阅读。

1. 将 `buy/sell/neutral` 改为“偏积极/中性观察/偏谨慎”。
2. 增加普通用户摘要。
3. 所有结论增加数据更新时间和数据完整度。
4. 新闻模块改名或隐藏不可靠来源。
5. 详情页增加“不构成投资建议”的克制提示。

### 第二阶段：建设自选股池智能参考

目标：让用户能从自己的自选股中看到重点机会和主要风险。

1. 新增“今日自选股参考”页面。
2. 新增“重点关注 / 继续观察 / 谨慎关注 / 数据不足”分组。
3. 基于本地因子、风险、策略、历史行情生成参考排序。
4. 每只股票给出支撑因素和风险因素。
5. 增加自选股整体风险概览。

### 第三阶段：增强风险识别

目标：让用户能理解为什么需要谨慎。

1. 增加风险雷达。
2. 将机构持仓改为重要股东变动。
3. 增加股东增减持解释。
4. 增加买入前和卖出前检查清单。
5. 将高风险股票优先推送到风险提醒区。

### 第四阶段：增强学习能力

目标：让用户在使用过程中逐步学会股票分析。

1. 每个指标增加“什么意思”和“为什么重要”。
2. 回测结果增加适用场景说明。
3. 增加案例式解释。
4. 增加组合风险分析。
5. 增加“我为什么这样判断”的解释链路。

## 11. 成功标准

普通用户版本的成功标准不是让用户每天交易更多，而是让用户：

- 少被单一指标误导。
- 少追高。
- 能识别明显风险。
- 能理解买卖前应该检查什么。
