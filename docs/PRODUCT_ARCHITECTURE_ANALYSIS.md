# AIStock 长期产品与商业化架构分析

## 1. 产品定位

从长期商业化角度，AIStock 不应定位为“荐股 App”，而应定位为：

> 面向普通投资者的股票学习、风险识别、自选股分析与决策辅助平台。

这个定位更适合当前项目已有能力，也更利于长期发展。

当前项目已经具备行情、历史数据、因子评分、风险提示、策略回测、分红、重要股东变动等基础能力。如果直接包装成“买卖推荐”，容易让普通用户误解，也容易放大合规风险。

更稳健的产品方向是：

- 帮用户理解股票。
- 帮用户识别风险。
- 帮用户比较自选股。
- 帮用户形成买卖前检查流程。
- 帮用户学习基本投资分析方法。
- 帮用户在自选股票池中找到更值得关注的对象。

核心不是替用户做决定，而是提高用户做判断的质量。

## 2. 合规与商业边界

长期考虑经济效益时，必须先明确产品边界。

如果产品以“推荐买入”“推荐卖出”“会员荐股”“承诺收益”为核心，就可能触碰证券投资咨询监管边界。更稳健的商业化方向是销售工具能力、数据整理能力、风险识别能力、学习能力和组合管理能力，而不是销售确定性收益或买卖指令。

建议坚持以下边界：

- 不输出“立即买入”“强烈卖出”“必涨”“稳赚”等表达。
- 不把历史回测包装成未来收益承诺。
- 不把自选股排序包装成确定性收益排名。
- 不把数据不足的结果包装成明确结论。
- 不替用户决定仓位和交易动作。
- 所有分析结果都明确标注“仅供参考，不构成投资建议”。

更合适的表达方式：

- 偏积极
- 中性观察
- 偏谨慎
- 数据不足，暂不判断
- 适合重点观察
- 风险较多，建议谨慎

商业化时，应把产品卖点设计为：

- 自选股风险识别。
- 普通语言解释。
- 分析报告。
- 组合风险管理。
- 投资学习辅助。
- 数据刷新和分析额度。

而不是：

- 今日牛股。
- 必买股票。
- 最佳买点。
- 保证收益。
- 跟单交易。

## 3. 产品分层架构

长期产品可以分为四层。

### 3.1 免费层：行情、自选股与基础学习

免费层用于获客和建立用户信任。

核心能力：

- 股票搜索。
- 实时行情。
- 自选股管理。
- 个股基础资料。
- 基础涨跌幅与价格趋势。
- 常见指标解释。
- 基础风险提示。

适合普通用户第一次使用时快速理解产品价值。

免费层不应给太多复杂分析，否则用户难以理解，也会削弱会员价值。

### 3.2 普通会员层：自选股智能分析

普通会员层应成为主要收入来源。

核心能力：

- 自选股健康度。
- 今日自选股参考。
- 重点关注 / 继续观察 / 谨慎关注 / 数据不足分组。
- 个股支撑因素。
- 个股风险因素。
- 数据完整度。
- 数据更新时间。
- 买入前检查清单。
- 卖出前检查清单。

这一层最符合普通用户的真实需求：用户已经有自己的自选股，但不知道哪些更值得关注、哪些风险更高。

### 3.3 高级会员层：组合与策略工具

高级会员层面向更有经验的用户。

核心能力：

- 自选股组合风险分析。
- 行业集中度分析。
- 单只股票暴露风险。
- 高风险股票数量统计。
- 回测结果解释。
- 策略适配度分析。
- 历史信号复盘。
- 分红稳定性分析。
- 重要股东变动趋势。

高级会员层的重点不是更多“结论”，而是更强的分析深度。

### 3.4 专业版 / B 端层：研究辅助

专业版可以面向投教机构、财经内容团队、小型研究团队或专业用户。

核心能力：

- 批量股票分析。
- 分析报告导出。
- 因子配置。
- 策略模板。
- 数据导出。
- 管理后台。
- 用户行为统计。
- 分析任务调度。

这一层可以作为长期商业化扩展，不应在早期过度投入。

## 4. 技术架构建议

长期架构不应继续把主要逻辑集中在单个后端入口文件中，而应拆成清晰的服务层。

推荐架构如下：

```text
数据源层
- EastMoney：实时行情、搜索、盘口类数据
- TuShare Pro：历史行情、财务、分红、重要股东、复权等
- 本地数据库：统一缓存、历史沉淀、分析结果存储

数据治理层
- 数据新鲜度判断
- 本地优先读取
- 缺失或过期才刷新
- 数据质量校验
- 数据完整度评分

分析引擎层
- 因子评分
- 风险评分
- 趋势评分
- 股东变动分析
- 分红稳定性分析
- 策略回测摘要
- 自选股综合排序

产品决策层
- 普通用户解释
- 自选股分组
- 风险标签
- 操作前检查清单
- 学习建议

接口层
- App API
- 管理后台 API
- 会员权限 API
- 数据任务 API

前端层
- 普通用户版
- 进阶分析版
- 管理后台
```

核心原则：

- 数据归数据。
- 分析归分析。
- 产品表达归产品表达。
- 权限归权限。
- 任务调度归任务调度。

这样后续无论是做会员版、Web 版、App 版、管理后台还是专业版，都不需要反复重写底层能力。

## 5. 后端模块拆分建议

建议逐步拆分为以下模块。

### 5.1 数据源服务

职责：

- 封装 EastMoney。
- 封装 TuShare Pro。
- 处理外部接口失败。
- 限制 TuShare Pro 调用频率。
- 统一返回结构。

注意：

- EastMoney 实时行情可以相对实时获取。
- TuShare Pro 数据应继续本地缓存优先。
- 不支持个股语义的新闻接口，不应包装成个股新闻。

### 5.2 数据缓存与新鲜度服务

职责：

- 判断本地数据是否存在。
- 判断本地数据是否过期。
- 判断是否需要刷新。
- 标记数据更新时间。
- 生成数据完整度。

这是产品信任感的基础。普通用户需要知道当前结论是基于新数据、旧数据还是不完整数据。

### 5.3 分析引擎服务

职责：

- 计算估值评分。
- 计算趋势评分。
- 计算波动风险。
- 计算盈利质量。
- 计算成长变化。
- 分析分红稳定性。
- 分析重要股东变动。
- 归纳策略回测结果。

分析引擎应尽量使用稳定规则，先保证可解释，再考虑复杂模型。

### 5.4 自选股推荐参考服务

职责：

- 读取用户自选股。
- 汇总本地分析结果。
- 输出重点关注、继续观察、谨慎关注、数据不足分组。
- 生成每只股票的支撑因素和风险因素。
- 给出自选股整体风险概览。

该服务是普通会员层的核心。

### 5.5 普通语言解释服务

职责：

- 将专业指标转换为普通用户能理解的表达。
- 解释为什么股票被归入某一类。
- 解释每个风险标签的含义。
- 解释回测和策略信号的适用边界。

产品的专业感不只来自数据，还来自解释质量。

### 5.6 权限与订阅服务

职责：

- 免费用户、普通会员、高级会员、管理员分层。
- 控制自选股数量。
- 控制每日分析次数。
- 控制批量分析能力。
- 控制高级报告访问。
- 记录会员状态。

这一层应尽早预留，避免后期商业化时大改接口。

### 5.7 审计与留痕服务

职责：

- 记录用户查看过哪些分析结果。
- 记录当时使用的数据版本和更新时间。
- 记录系统输出的风险提示。
- 记录用户是否点击检查清单。

这对合规、安全、问题追踪和产品优化都有价值。

## 6. 核心产品模块

长期最值得建设的产品模块包括以下六个。

### 6.1 自选股机会雷达

商业价值最高，适合作为会员核心功能。

能力：

- 每日自选股参考。
- 自选股排序。
- 重点关注股票。
- 谨慎关注股票。
- 数据不足股票。
- 最近变化提醒。

输出方式应克制，不输出直接买卖指令。

### 6.2 个股风险说明书

目标是帮助普通用户看懂一只股票的主要风险。

能力：

- 估值风险。
- 波动风险。
- 业绩风险。
- 股东减持风险。
- 分红不稳定风险。
- 数据不足风险。
- 新闻或事件风险。

每个风险都需要解释“为什么重要”。

### 6.3 买卖前检查清单

目标是帮助用户形成操作纪律。

买入前检查：

- 是否了解公司主营业务。
- 估值是否偏高。
- 业绩是否稳定。
- 是否存在重要股东减持。
- 是否刚经历大幅上涨。
- 下跌 10%-20% 是否能承受。
- 是否持仓过度集中。

卖出前检查：

- 买入逻辑是否失效。
- 业绩是否恶化。
- 估值是否过高。
- 重要股东是否连续减持。
- 是否只是短期波动导致恐慌。

系统可以提示通过项和未通过项，但不替用户下交易命令。

### 6.4 组合风险管家

目标是从单只股票分析升级到组合分析。

能力：

- 行业集中度。
- 单只股票集中度。
- 高风险股票数量。
- 同类股票重复持仓。
- 整体波动风险。
- 组合数据完整度。

普通用户常见问题不是不会选一只股票，而是不知道自己的整体风险已经过高。

### 6.5 学习式分析报告

目标是提高用户留存和学习价值。

能力：

- 一句话结论。
- 三个支撑因素。
- 三个风险因素。
- 关键指标解释。
- 为什么这样判断。
- 下一步应该观察什么。

报告应强调“学习和参考”，而不是“指令和承诺”。

### 6.6 数据健康中心

目标是提高产品可信度。

能力：

- 数据是否完整。
- 数据是否过期。
- 哪些数据来自 EastMoney。
- 哪些数据来自 TuShare Pro。
- 哪些结论因数据不足而降级。
- 最近一次刷新时间。

长期看，数据健康中心是专业产品和普通工具的分界线。

## 7. 数据策略

### 7.1 本地优先

后端数据策略应继续坚持：

1. 先查本地数据库。
2. 判断数据是否存在。
3. 判断数据是否过期。
4. 未过期直接使用本地数据。
5. 缺失或过期才访问外部接口。
6. 外部接口结果写回本地数据库。
7. 分析结果基于本地数据生成。

这个策略对成本、稳定性、访问限制和用户体验都很重要。

### 7.2 实时与非实时分层

不同数据应有不同刷新策略：

| 数据类型 | 推荐来源 | 刷新策略 |
|---|---|---|
| 实时行情 | EastMoney | 可实时刷新 |
| 股票搜索 | EastMoney | 可实时查询 |
| 历史行情 | TuShare Pro / 本地 | 本地优先，按交易日刷新 |
| 财务因子 | TuShare Pro / 本地 | 本地优先，按报告期刷新 |
| 分红 | TuShare Pro / 本地 | 本地优先，低频刷新 |
| 重要股东 | TuShare Pro / 本地 | 本地优先，按报告期刷新 |
| 复权因子 | TuShare Pro / 本地 | 本地优先，按需要刷新 |
| 自选股分析结果 | 本地计算 | 数据变化后重新计算 |

### 7.3 数据完整度

每个分析结果都应带上数据完整度。

示例：

- 完整：关键数据齐全，结论可信度较高。
- 基本完整：缺少部分辅助数据，结论可参考。
- 不完整：缺少关键数据，只能弱参考。
- 不足：不应形成明确判断。

- 个股风险说明书。
- 买卖前检查清单。
- 每日自选股参考。

### 8.3 高级会员能力

- 组合风险分析。
- 策略适配度。
- 回测解释。
- 重要股东变动趋势。
- 分红稳定性分析。

### 8.4 专业版能力

- 策略模板。
- 因子配置。
- 数据导出。
- 团队管理。

### 8.5 可限制的商业化指标

不建议用“更强买卖信号”作为付费点。

## 9. 前端产品形态

### 9.1 普通用户首页

首页应优先展示：

- 今日自选股参考。
- 重点关注。
- 谨慎关注。
- 数据不足。
- 整体风险水平。
- 最近数据更新时间。

首页不是信息堆砌页，而应该是“今天该看什么”的入口。

### 9.2 个股详情页

个股详情页顶部应展示：

- 一句话摘要。
- 当前参考状态。
- 主要支撑因素。
- 主要风险因素。
- 数据完整度。
- 数据更新时间。

复杂指标应放在下方，避免普通用户一进入页面就被专业数据淹没。

### 9.3 自选股分析页

自选股分析页应支持：

- 按综合参考排序。
- 按风险排序。
- 按数据完整度排序。
- 按最近变化排序。
- 切换重点关注、继续观察、谨慎关注、数据不足。

### 9.4 学习页

学习页不应做成纯文章库，而应结合用户正在看的股票。

例如：

- 什么是估值偏高。
- 为什么股东减持需要关注。
- 最大回撤是什么意思。
- 为什么不能只看涨跌幅。
- 为什么数据不足时不能下结论。

## 10. 分阶段路线

### 第一阶段：数据与表达打底

目标：让当前项目更稳、更可信。

重点：

- 坚持本地优先数据策略。
- 区分 EastMoney 与 TuShare Pro 数据。
- 增加数据更新时间。
- 增加数据完整度。
- 将买卖信号改成更克制的参考表达。
- 修正“机构持仓”为“重要股东变动”语义。

### 第二阶段：普通用户版核心功能

目标：形成可收费的普通会员核心能力。

重点：

- 今日自选股参考。
- 自选股机会雷达。
- 自选股健康度。
- 支撑因素与风险因素。
- 买卖前检查清单。
- 个股风险说明书。

### 第三阶段：会员体系

目标：让产品具备商业化基础。

重点：

- 免费 / 普通会员 / 高级会员 / 管理员权限。
- 分析次数限制。
- 自选股数量限制。
- 报告访问权限。
- 订阅状态。
- 支付接入预留。

### 第四阶段：组合与策略能力

目标：提高留存和高级会员价值。

重点：

- 组合风险分析。
- 行业集中度。
- 策略适配度。
- 回测解释。
- 历史信号复盘。
- 分红稳定性分析。

### 第五阶段：专业版扩展

目标：探索更高客单价收入。

重点：

- 批量分析。
- 数据导出。
- 报告导出。
- 因子配置。
- 策略模板。
- 管理后台。

## 11. 成功指标

产品成功不应只看用户是否交易更多，而应看用户是否更理性地使用工具。

建议关注：

- 自选股添加率。
- 自选股分析页打开率。
- 风险提示点击率。
- 检查清单完成率。
- 会员转化率。
- 会员续费率。
- 用户查看数据完整度的比例。
- 用户是否持续回来看自选股变化。
- 高风险股票提醒后的用户行为。

长期看，真正有价值的指标是：

- 用户是否减少盲目追高。
- 用户是否能理解风险。
- 用户是否愿意持续使用产品做决策前检查。
- 用户是否认为产品帮助自己建立了分析框架。

## 12. 总结

AIStock 如果要长期产生经济效益，最稳健的方向不是做激进荐股软件，而是做：

> 数据工具 + 风险识别 + 投资学习 + 自选股决策辅助 + 会员制分析能力。

技术架构上，应围绕“本地优先数据治理、可解释分析引擎、自选股智能参考、权限订阅体系、审计留痕”来设计。

产品架构上，应从免费行情和基础学习切入，以自选股机会雷达和风险说明书作为会员核心价值，再逐步扩展组合风险、高级策略和专业版能力。
