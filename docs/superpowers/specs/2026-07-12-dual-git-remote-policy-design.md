# Gitee 主远端与 GitHub 辅助同步策略

## 目标

将 Gitee 作为日常代码提交的主远端，将 GitHub 作为辅助镜像。每次需要发布已验证的提交时，两个远端都必须更新，避免长期分叉。

## 持久化范围

- 在项目根目录 `AGENTS.md` 中记录项目级规则，使规则随仓库保留。
- 在本机 Codex 全局 `AGENTS.md` 中记录相同规则，使后续项目和会话默认遵循该策略。

## 远端约定

- `gitee`：`https://gitee.com/caweinhen/aistock.git`，主远端。
- `origin`：`https://github.com/caweinchen/aistock.git`，辅助远端。
- 默认发布分支为 `main`，除非用户明确指定其他分支。

## 同步流程

1. 本地验证通过并形成待发布提交。
2. 获取两个远端的目标分支，检查提交包含关系，禁止未经确认的强制覆盖。
3. 先将提交推送到 Gitee。
4. 再将同一提交推送到 GitHub。
5. 重新读取两个远端分支，确认它们指向相同提交；只有两端均成功才报告同步完成。

## 异常处理

- 任一远端认证或网络失败时，保留已经成功的结果并明确报告未同步的远端，不将任务标记为全部完成。
- 两端历史发生分叉时，停止自动推送并报告提交差异，未经用户确认不得强制推送。
- 不覆盖本地未提交修改，不为同步操作改写无关提交历史。

## 验收标准

- 项目级和全局规则文件均包含明确的 Gitee 主、GitHub 辅策略。
- 当前项目保留 `gitee` 和 `origin` 两个正确的远端地址。
- 后续发布操作按 Gitee 先、GitHub 后执行，并验证两个远端最终提交一致。
