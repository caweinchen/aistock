## 目标与边界

在仓库根目录新增独立的 `pvz-game/` React + Vite 应用，交付一关可完整游玩的原创卡通庭院塔防小游戏。它与现有 `frontend/` 股票应用并列，拥有独立依赖、构建和测试命令，不导入、不修改现有应用代码。

首版包含 5×9 草坪、阳光经济、三种植物、两种敌人、分波进攻、铲子、暂停、声音开关、胜负结算和重新开始。不包含关卡选择、存档、联网、付费、多人模式或原作素材。

## 技术结构

`pvz-game/src/game/` 保存纯 TypeScript 领域模型、配置和状态转换；`src/components/` 保存 React 控件和棋盘单位；`src/assets/` 保存原创位图；`src/styles/` 保存布局、动画和响应式规则。

游戏引擎以固定时间步接收 `tick`，返回新状态。放置、铲除、收集阳光、暂停与重新开始均为显式 action。React 只负责派发 action 和渲染状态，避免计时器与 DOM 成为规则真相。浏览器失焦时暂停，恢复时不补算离线时间。

核心状态包括：

- `phase`: playing、paused、won 或 lost
- `sun`: 当前阳光
- `selectedTool`: 植物卡、铲子或空
- `plants`: 所在行列、类型、生命、生产/攻击冷却
- `zombies`: 所在行与连续横坐标、类型、生命、移动/攻击状态
- `projectiles` 与 `sunDrops`: 位置、归属和剩余时间
- `wave`: 已生成数量、存活数量、下一次生成时间与最终波标记

## 玩法规则

玩家初始拥有 150 阳光。向日葵消耗 50 并周期产出阳光，豌豆射手消耗 100 并攻击本行最前方敌人，坚果消耗 75 并提供高耐久阻挡。植物卡放置后进入各自冷却；阳光不足、冷却未结束或格子占用时不扣资源并给出可见反馈。

普通敌人按标准速度和生命推进，路障敌人更耐打。敌人与植物接触后停止移动并周期啃咬，植物被消灭后继续推进。投射物只命中本行首个相交敌人。敌人越过草坪左边界立即判负；最终波全部生成且场上无存活敌人时判胜。胜负后停止模拟，结算层提供重新开始。

数值集中在配置文件中，测试使用同一配置或显式测试配置，不把平衡参数散落在组件内。

## 界面与视觉

首屏直接显示游戏。顶栏包含阳光计数、波次进度、暂停和声音图标按钮；桌面端植物卡位于棋盘左侧，窄屏移至棋盘上方；铲子作为独立工具按钮。中央棋盘始终保持 9:5 网格比例，单位和动态内容不得改变格子尺寸。

视觉采用明亮庭院、木质工具栏、绿色草坪和高对比危险提示组成的多色卡通风格。植物、敌人、背景和阳光使用为本项目生成的原创位图，不使用《植物大战僵尸》原作角色图、商标或提取素材。生命受损、卡片冷却、选中、不可放置、命中和啃咬均有清晰状态变化。

桌面和横屏为主要游玩形态。窄屏保持整块棋盘可见并允许页面纵向容纳工具栏，不使用遮挡棋盘的悬浮说明。交互按钮具备可访问名称和键盘焦点；尊重 `prefers-reduced-motion`。

## 声音与异常处理

声音默认开启，但首个用户手势前不自动播放；声音开关可静音全部效果。资源加载失败时仍显示颜色与文字/形状组合的可玩降级界面。重复点击阳光或格子必须幂等，任何无效放置不得消费阳光。窗口恢复、掉帧或超长 tick 使用最大步长限制，避免单位瞬移穿越碰撞。

## 测试与验收

Vitest 覆盖初始化、合法/非法放置、资源扣除、冷却、产阳光、射击命中、啃咬、铲除、刷怪、暂停、胜负和重开。测试先失败再实现对应规则。

Playwright 覆盖完整用户路径：选择并放置植物、收集阳光、铲除植物、暂停/恢复、触发胜负并重新开始；同时在桌面和手机视口检查可操作性、文字溢出和遮挡。最终验收还包括 TypeScript 检查、生产构建、浏览器控制台无错误，以及桌面/手机截图和关键位图像素非空检查。

## 完成标准

- `pvz-game/` 可独立安装、测试、启动和构建。
- 一关从开局到胜利或失败的闭环可操作。
- 已批准范围内的规则均有自动化测试。
- 桌面与窄屏无关键控件遮挡或文字溢出。
- 原创资源正常加载，未引入原作受版权保护素材。
- 未改变现有股票应用行为。
